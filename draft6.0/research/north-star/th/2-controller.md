# 2 — prefrontal cortex / working memory / ตัวประสานงาน

> คำถามของคุณ: *"prefrontal cortex ควรหน้าตาเป็นยังไง? ใช้โมเดลไหน? แล้วชิ้นส่วนต่างๆ สื่อสารกันยังไง?"* คำตอบที่ลู่เข้าหากัน: **ไม่ใช่ตาข่ายใหญ่ก้อนเดียว** แต่เป็น *ตัวประสานงานที่เล็กและจำกัดแบนด์วิดท์* ซึ่งถือของไว้ทำงานนิดเดียว ไป query ส่วนที่เหลือจากหน่วยความจำ แล้ว broadcast บทสรุปกลับไปให้พวกผู้เชี่ยวชาญ ความขาดแคลนนี่แหละคือประเด็น — มันบังคับให้แต่ละส่วนแตกความชำนาญและเห็นพ้องกัน

---

## Global Workspace Theory (GWT) — ประสาทวิทยาของ "นิดเดียว แต่กระจายไปทั่ว"

*Baars, 1988; Global neuronal workspace: Dehaene & Changeux, 2011 ([Neuron 70:2, 200-227](https://www.sciencedirect.com/science/article/pii/S0896627311003060)).*

### ปัญหาที่มันแก้

ในปี 1988 Baars นักประสาทวิทยาถามคำถามที่ฟังดูปรัชญามาก: *"ทำไมสมองที่ประมวลผลขนาน 100 พันล้านเซลล์จึงรู้สึกเหมือนมี 'ตัวตน' เดียวที่กำลังคิดทีละอย่าง?"*

คำตอบของเขาไม่ใช่เรื่องจิตวิญญาณ แต่เป็นวิศวกรรม: มันเป็นแบบนั้นเพราะ **มีแบนด์วิดท์จำกัดตามการออกแบบ**

### กลไกจริงๆ

GWT บอกว่าสมองคือ **ฝูงของผู้เชี่ยวชาญที่ทำงานขนาน** — visual cortex วิเคราะห์ภาพ, auditory cortex ฟังเสียง, hippocampus ดึงความจำ, motor cortex วางแผนการเคลื่อนไหว ทั้งหมดทำงานพร้อมกันและ "ไม่รู้ตัว" ของกันและกัน

แต่มีโครงสร้างพิเศษที่เรียกว่า **global workspace** (ในสมองจริง: prefrontal + parietal cortex + long-range connections) ที่ทำหน้าที่แตกต่างออกไป:

1. **Competition**: ผู้เชี่ยวชาญทุกตัวส่ง "ใบสมัคร" เข้า workspace โดยการ activate neurons ที่เชื่อมต่อกับมัน ตัวไหนชนะการแข่งขัน (มีสัญญาณแรงกว่า มีความเกี่ยวข้องมากกว่า) ก็ได้เข้า workspace
2. **Ignition**: เมื่อ workspace รับสิ่งใดสิ่งหนึ่ง มันเกิด "ignition" — การ synchronize ของ neurons ขนาดใหญ่ทั่วสมอง ซึ่งในการทดลอง EEG/MEG เห็นเป็นคลื่น P3 (ประมาณ 300ms หลังเห็น stimulus ที่มีนัยสำคัญ)
3. **Broadcast**: สิ่งที่เข้า workspace ถูก broadcast กลับออกไปยังผู้เชี่ยวชาญ **ทุกตัว** พร้อมกัน ทุกตัวได้ยิน "ข่าว" เดียวกัน แล้วแต่ละตัวตอบสนองตามความชำนาญของตัวเอง

Capacity ของ workspace: คลาสสิกคือ **Miller's magic number ~7 ±2** สิ่งของ ในทางทดลองสมัยใหม่ ~3-4 สิ่ง ซึ่งต่ำมากเมื่อเทียบกับการประมวลผลทั้งหมดที่เกิดขึ้น

หลักฐานทางประสาทวิทยา: ผู้ป่วยที่มี lesion ใน prefrontal มี "awareness" ต่ำลง แต่ยัง process ข้อมูลได้ (blindsight) ยืนยันว่า workspace แยกออกจาก processing

### ผลที่น่าตื่นเต้น

GWT ทำนายปรากฏการณ์หลายอย่างที่พิสูจน์ได้:
- **Attentional blink**: เมื่อเห็นสองสิ่งห่างกัน <500ms ในสตรีมเร็ว สิ่งที่สองหายไป *(workspace ยังไม่ว่าง)*
- **Inattentional blindness**: ไม่เห็น gorilla ในวิดีโอ *(กอริลลาไม่ชนะการแข่งขัน)*
- **Binocular rivalry**: ตาซ้าย-ขวาเห็นต่างกัน สมองสลับไปมา *(การ compete เข้า workspace)*

### สำหรับเรา

GWT คือ **template ของ "วิธีที่สมองของเฟส 2 ทำงาน"** และที่สำคัญที่สุดคือบทเรียนที่ counter-intuitive:

**ความจำกัดของ bandwidth คือ feature ไม่ใช่ bug** — มันบังคับให้เกิด "selection" ว่าอะไรสำคัญที่สุดในขณะนี้ ถ้าทุกอย่างเข้า workspace พร้อมกัน ไม่มีอะไรถูกเลือก ไม่มีการประสานงาน

สำหรับชิป: บัสระหว่าง SCFF block กับ prefrontal module **ไม่ควรกว้าง** — ยิ่งแคบยิ่งดี (และถูกกว่า ประหยัดพลังงานกว่า) การ competition เข้า workspace เป็น **winner-take-all** ซึ่งคุณมีเป็น primitive อยู่แล้ว การ broadcast เป็น **หนึ่ง-ต่อ-หลาย** ซึ่งถูกมากบน analog bus

## Coordination Among Neural Modules Through a Shared Global Workspace — เวอร์ชันที่สร้างได้จริง

*Goyal, Didolkar, Lamb, …, Mozer, Blundell, Bengio, ICLR 2022 ([arXiv 2103.01197](https://arxiv.org/abs/2103.01197)).*

### ปัญหาที่มันแก้

GWT เป็นทฤษฎีที่สวยงาม แต่สร้างเป็นโมดูล deep learning ได้ยังไง? เปเปอร์นี้ทำให้ GWT เป็น **building block** ที่เสียบเข้ากับ neural network ได้ทันที

### กลไกจริงๆ

สถาปัตยกรรมมีสองส่วน:

**ส่วนที่ 1 — Specialist modules M₁, M₂, ..., Mₖ**: แต่ละโมดูลอาจเป็น LSTM เล็กๆ, transformer block เล็กๆ, หรืออะไรก็ได้ที่รับ input ผลิต representation แต่พวกมัน **ไม่สื่อสารกันโดยตรง** ทั้งหมดก็แค่มีส่วน output ไปยัง workspace

**ส่วนที่ 2 — Shared workspace W** มีขนาดเล็กมาก (เช่น 2-4 slots ขนาด d-dimension) แต่ละ slot เป็น vector:

1. ทุกโมดูล M_i ผลิต query q_i พร้อม value v_i
2. **Competition**: คำนวณ attention score ของแต่ละโมดูลต่อ slot ปัจจุบัน: score_i = q_i · W_slot
3. softmax(score) → เลือก top-k โมดูลที่ชนะ write เข้า slot (หรือ soft attention ก็ได้)
4. **Broadcast**: เนื้อหาใน workspace ถูกส่งกลับให้ทุกโมดูล M_i อ่าน (cross-attention)

เปเปอร์แสดงให้เห็นว่า bottleneck แบบนี้บังคับให้เกิด **modularity เองตามธรรมชาติ**: โมดูลแต่ละตัวต้อง specialize เพราะมันไม่สามารถเอาทุกอย่างขึ้นไปบน workspace พร้อมกัน ตัวที่จะชนะคือตัวที่มี "สิ่งที่ workspace ต้องการในขณะนี้" เท่านั้น

### ผลที่น่าตื่นเต้น

บน systematic generalization tasks (เช่น SCAN, COGS — งานที่ต้องเอา composition ของ rules มาใช้กับ input ใหม่):
- Transformer ธรรมดา: ล้มเหลวใน out-of-distribution combinations
- Shared Global Workspace: generalize ข้ามได้ดีกว่าอย่างมีนัยสำคัญ

เหตุผล: workspace บังคับให้ modules encode ข้อมูลในรูปที่ "compact + composable" — เพราะมันต้องเขียนลง slot ขนาดเล็ก ไม่ใช่ vector ยักษ์

### สำหรับเรา

นี่คือคำตอบที่เป็นรูปธรรมของ *"สอง/หลายสมองสื่อสารกันยังไง?"*:

**workspace ที่ใช้ร่วมกัน + bandwidth ceiling + broadcast** — ไม่ใช่ "เดินสาย crossbar ทุก-ต่อ-ทุก" ซึ่งในแอนะล็อกแปลว่า:

- SCFF blocks แต่ละกลุ่ม = specialist modules (M_i)
- Prefrontal module = workspace W (register file เล็กๆ ไม่กี่ slot)
- การแข่งชิง slot = WTA circuit (มีอยู่แล้วใน substrate)
- Broadcast = analog bus กว้างแต่สั้น (signal ไปทั่วแต่ไม่ผ่าน crossbar)

ข้อที่ต้องตัดสินใจในเฟส 2: จำนวน slot ใน workspace — 2 slot? 4 slot? และขนาดของ slot vector (จำนวน bits หรือ weight lines) มีผลต่อ bottleneck severity โดยตรง

## Recurrent Independent Mechanisms (RIMs) — recurrence แบบแยกโมดูล ที่จะทำงานเฉพาะตอนเกี่ยวข้อง

*Goyal, Lamb, Hoffmann, Sodhani, Levine, Bengio & Schölkopf, 2019 ([arXiv 1909.10893](https://arxiv.org/abs/1909.10893)).*

### ปัญหาที่มันแก้

LSTM ก้อนใหญ่ก้อนเดียวมีปัญหา: มันต้อง update **ทุก unit** ทุก time step ไม่ว่าจะเกิดอะไรขึ้น ซึ่งไม่สมเหตุสมผล — ถ้าคุณกำลังฟังเพลงและเห็น notification โผล่ ระบบ auditory ไม่ต้องหยุดทำงาน แค่ระบบ visual ตอบสนองส่วน RNN ธรรมดาทุก unit update พร้อมกันทุกครั้ง RIMs แก้นี้

### กลไกจริงๆ

RIMs ประกอบด้วย **K หน่วย recurrent ขนาดเล็ก** (เช่น K=6 LSTM ขนาด 100 unit แทน LSTM เดียว 600 unit) แต่ละหน่วยมี hidden state ของตัวเอง

ในแต่ละ time step:
1. คำนวณ **attention score** ของแต่ละหน่วย i ต่ออินพุต x_t: score_i = q_i · k_x (linear projection)
2. **เลือก top-k = 2 หน่วย** (กลุ่มที่เกี่ยวข้องที่สุด) หน่วยที่เหลือ **ถูก freeze** ทั้งหมด — hidden state ไม่เปลี่ยน ไม่มีการคำนวณ
3. **Active units update**: เฉพาะ k=2 หน่วยที่ถูกเลือก update hidden state ตาม LSTM equations พร้อมรับอินพุต
4. **Communication**: active units อ่านจาก inactive units ได้ผ่าน attention แต่ไม่แก้พวกมัน (สื่อสารแต่ไม่รบกวน)
5. เอาต์พุตรวมมาจาก active units

ผลคือ: แต่ละหน่วยแตกความชำนาญไปเป็น "กลไกหนึ่งของโลก" — อาจมีหน่วยที่เชี่ยวชาญ motion, อีกหน่วยเชี่ยวชาญ colour, อีกหน่วยเชี่ยวชาญ object identity และแต่ละหน่วยตอบสนองเฉพาะตอนที่ความชำนาญของมันถูกเรียก

### ผลที่น่าตื่นเต้น

บน **bouncing balls task** (จำนวน balls แตกต่างกัน physics ต่างกัน): RIMs แต่ละหน่วยเรียนรู้ track ball อันเดียว หน่วยที่เกี่ยวข้องกับ ball อันใดอัน activate เมื่อมัน collide เท่านั้น RNN เดียวก้อนใหญ่: ทุก unit update ทุกครั้ง ไม่มี specialization

บน **copy task** (จำ sequence ยาวแล้ว reproduce): RIMs ที่เก็บแต่ละ element ใน unit แยกกันทำได้ดีกว่า LSTM เดียวมาก

Systematic generalization บน SCAN: RIMs ทำได้ดีกว่า LSTM ธรรมดา 15-20%

### สำหรับเรา

RIMs คือวิธีหลีกเลี่ยง "LSTM เต็มตัวที่รู้ทุกอย่างพร้อมกัน" ที่คุณกังวล และมันเหมาะกับ substrate คุณอย่างยิ่ง:

**โมดูลที่หลับ = ไม่ใช้พลังงาน** — บน analog ถ้าหน่วยที่ไม่ถูกเลือกไม่มี clock หรือ charge cycle มันไม่บริโภคพลังงาน นี่คือประสิทธิภาพแบบ biological จริงๆ (สมองใช้พลังงานน้อยกว่า 20W แต่มีสิบพันล้านเซลล์เพราะแต่ละเซลล์ fire น้อยมาก)

**SCFF blocks = RIM units**: แต่ละ block แตกความชำนาญจาก unsupervised training เฉพาะ block ที่เกี่ยวข้องกับ input ปัจจุบัน activate ที่เหลือ freeze ซึ่งในชิปหมายถึง "ไม่สับเปลี่ยนประจุ" ซึ่งถูกมาก

**top-k selection ≈ WTA + threshold**: เลือก 2 จาก 6 คือ WTA ที่มีขั้นบันไดสอง ซึ่ง implement ด้วย analog comparator cascade ได้

## The Consciousness Prior — ความคิดที่มีมิติต่ำ

*Bengio, 2017 ([arXiv 1709.08568](https://arxiv.org/abs/1709.08568)).*

### ปัญหาที่มันแก้

Deep learning ทำงานใน high-dimensional space — hidden layer อาจมีหมื่น dimensions แต่ถ้าถามคนว่า "กำลังคิดอะไรอยู่?" เขาพูดออกมาเป็นประโยคได้ ซึ่งมีตัวแปรไม่กี่สิบอัน Bengio ถามว่า: *"ทำไม? และมันมีนัยสำคัญสำหรับ deep learning ยังไง?"*

### กลไกจริงๆ

Bengio เสนอว่า **conscious thought** คือ "low-dimensional projection" ของ high-dimensional unconscious representations กลไก:

1. มี **unconscious representation space** มิติสูง (เช่น hidden state h ขนาด 10,000)
2. **Consciousness prior** คือ attention mechanism ที่ project h ไปยัง **thought z** ขนาดเล็ก (เช่น 10-50 dimensions) โดยเลือก "ตัวแปรที่สำคัญที่สุดในตอนนี้" ผ่าน sparse attention
3. z คือ **ประโยคความคิด**: "วัตถุ A อยู่ในตำแหน่ง B", "สิ่งนั้นน่าจะนำไปสู่ C" — มีไม่กี่ตัวแปรแต่มีพลังมากเพราะเป็น abstraction

Hypothesis หลัก: **representation ที่มีประโยชน์ของโลกอยู่ใน low-dimensional thought space** ไม่ใช่ใน high-dimensional pixel space ดังนั้น:
- การเรียนรู้ควรผ่านคอขวดแคบๆ นั้น (information bottleneck)
- การ communicate ระหว่าง modules ควรผ่าน z ไม่ใช่ h โดยตรง
- การทำนายใน z-space ได้ generalization ที่ดีกว่า input-space

### ผลที่น่าตื่นเต้น

Bengio เสนอ architecture ที่ attention mechanism เลือก "ตัวแปร high-level ไม่กี่ตัว" จาก high-dim state แล้วเทรน z ให้สอดคล้องกัน บน visual reasoning tasks z ที่ sparse กว่าให้ OOD generalization ที่ดีกว่า เพราะ z บังคับให้เข้ารหัสใน "ภาษา" ที่โลกชอบ

### สำหรับเรา

Prior นี้ **ย้ำให้ workspace แคบเป็น learning constraint ไม่ใช่แค่ bandwidth limit**:

สิ่งที่ถูกถือไว้ใน prefrontal workspace ของเราควรเป็น **คลัสเตอร์ SCFF feature vector ไม่กี่ตัว** (centroid ของ pattern) ไม่ใช่ raw activation ทั้งหมด การ project หาก่อน แล้ว communicate จึงเป็นทั้งถูกกว่า (ส่งข้อมูลน้อยลง) และดีกว่า (บังคับ abstraction)

ในเชิงปฏิบัติ: ถ้า SCFF block แต่ละอันมีขนาด 100 unit เราไม่ต้องส่งทั้ง 100 ขึ้น workspace แค่ส่ง centroid 8-16 dimensions ก็พอ นั่นคือ factor 6-12x ประหยัดใน bus

## PBWM — gating: อะไรที่ได้เข้า working memory และเมื่อไหร่ที่ปล่อยมันไป

*O'Reilly & Frank, 2006, Neural Computation ("Prefrontal cortex and Basal-ganglia Working Memory").*

### ปัญหาที่มันแก้

Working memory มีปัญหาที่ยากมาก: ถ้ามันอัปเดตตลอดเวลา มันก็ถูก distract ง่าย แต่ถ้ามันเก็บทุกอย่างไว้ มันก็ไม่ยอมรับข้อมูลใหม่ต้องมีกลไกที่ตัดสินว่า **"เมื่อไหร่ควร update? เมื่อไหร่ควรล็อคไว้?"** และกลไกนั้นต้องเรียนรู้ได้ (ไม่ใช่กฎตายตัว) PBWM ตอบด้วยการโยน problem นี้ให้กับ basal ganglia

### กลไกจริงๆ

ในสมอง prefrontal cortex (PFC) เก็บ working memory ใน **stripes** — กลุ่มของ neurons ที่สามารถถือค่าไว้ผ่าน recurrent excitation แต่ basal ganglia (BG) ทำหน้าที่ **gate**:

สมองมี dopamine neuron ที่ fire แบบ **temporal difference (TD) error**: ถ้าผลลัพธ์ดีกว่าที่คาด → dopamine spike; ถ้าแย่กว่าที่คาด → dopamine dip ซึ่ง BG ใช้ TD error นี้ train gate ว่า:
- **D1 pathways (Go)**: ถ้า gate เข้า PFC แล้วผลดี → reinforce การเปิด gate (update WM)
- **D2 pathways (No-Go)**: ถ้า gate เข้า PFC แล้วผลแย่ → reinforce การปิด gate (ล็อค WM ไว้)

ผลคือ BG เรียนรู้ว่า "ข้อมูลประเภทไหน ในสถานการณ์ไหน ควรเข้า WM" — เป็นนโยบายการ gating ที่เรียนรู้ได้ทั้งหมด

กลไกเดียวกันในทิศตรงข้าม: เรียนรู้ว่า "เมื่อไหร่ควร clear WM" (ปล่อยของเก่าออก เตรียมรับใหม่)

### ผลที่น่าตื่นเต้น

PBWM จำลอง **Wisconsin Card Sorting Test** — งาน cognitive ที่บอกว่า rule เปลี่ยนกะทันหัน (ของเก่าผิดหมด) และต้อง adapt เร็ว: โมเดลที่ไม่มี gating: ช้าในการปรับตัว PBWM: adapt ได้เร็วคล้ายมนุษย์ ผู้ป่วย PFC damage: adapt ช้ากว่ามนุษย์ปกติ — consistent กับ model ที่ gate ผิดพลาด

PBWM ยัง explain **delayed match-to-sample task** (จำสีที่เห็นไว้ 2 วินาที แล้วตอบว่าคู่กับตัวไหน) ซึ่งต้องการ WM ที่ stable ระหว่าง delay โดยไม่ถูก distract

### สำหรับเรา

นี่คือ **พ่อแม่ทางชีววิทยาของ LSTM gate** และมันตอบคำถามออกแบบที่เป็นรูปธรรม: วงลูป prefrontal ของเฟส 2 ต้องมี **gate ที่เรียนรู้ได้** ตัดสินว่า:

1. อะไรได้เข้า workspace (Write gate)
2. อะไรถูกล้างออก (Clear gate)
3. เมื่อไหร่ควรคง workspace ไว้ขณะ query (Hold gate)

บน substrate: gate เป็น **เซลล์เล็กๆ ที่เทรนด้วยสัญญาณ TD error** — ซึ่งก็คือ "ความรู้สึกว่าถูก" ที่ critic ผลิตใน `4-signal.md` นั้นเอง gate ของ LSTM (forget, input, output) คือ approximation ราคาถูก; PBWM คือแบบจำลองที่มีหลักการกว่า ในแอนะล็อก gate เป็นแค่ transmission gate (transistor เดียว) ที่ถูกควบคุมด้วย voltage ที่เรียนรู้ได้

## รูปร่างของคำตอบ (ไฟล์นี้)

prefrontal cortex สำหรับเรา **ไม่ใช่โมเดล — มันคือตัวประสานงาน**:

- **Workspace ขนาดเล็ก (GWT / Shared-GW)** — รับสิ่งเดียวทีละครั้ง broadcast ออกไป, bandwidth จำกัดโดยตั้งใจ
- **Thought ที่ sparse มิติต่ำ (Consciousness Prior)** — ถือตัวแปรไม่กี่ตัวที่ทรงพลัง ไม่ใช่ raw features ทั้งหมด
- **Module recurrent ที่ sparse และตื่นเฉพาะตอนเกี่ยวข้อง (RIMs)** — แต่ละ block เชี่ยวชาญ เปิดแค่ตอนถูกเรียก
- **Gate ที่เรียนรู้ได้ (PBWM)** — เรียนรู้ว่าอะไรควรเข้าออก workspace จาก TD signal ไม่ใช่กฎตายตัว
- **Communication = competition + broadcast** ไม่ใช่ full crossbar ระหว่างทุก module

ความแคบของ workspace คือ inductive bias ของการเรียนรู้ และมันถูกมากในเชิงแอนะล็อก (WTA + analog bus) เทียบกับ alternative (full mesh)
